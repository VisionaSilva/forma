# TASTE.md — Terminal Green

## Philosophy
Root access granted. Proceed with intention. This is not a theme — it's a terminal session with design taste.

## What It Is
- Pure monospace energy. JetBrains Mono everywhere — headings, body, labels, code. The monospace grid IS the design.
- Green is signal, not decoration. Use it for active states, interactive elements, emphasis. When everything is green, nothing is.
- Zero border radius. Terminals don't have rounded corners. Every pixel of radius costs credibility.
- Information-dense. If a component feels spacious, tighten it. Maximize data per pixel.
- Hard corners, bright green on near-black. That's the whole palette.

## What It Isn't
- Not luxury (use Obsidian for that)
- Not friendly or warm (use Coral Bloom)
- Not neutral SaaS (use Soft Carbon)
- Not for non-technical audiences

## Rules
1. Green accent `oklch(0.80 0.22 145)` is reserved for: interactive elements, success states, focus rings, emphasis. Not backgrounds, not decoration.
2. Dimmer green `oklch(0.52 0.08 145)` for secondary content — not gray. Keep the monochromatic feel.
3. Base font size is 14px, not 16px — mono reads larger than proportional at same size.
4. Never use italic — it looks wrong in mono. Use color for emphasis.
5. Letter-spacing: 0 for body, 0.05em for all-caps labels.
6. Border radius: 0px everywhere. The only exception: circular status dots.
